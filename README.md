# Distributed Job Scheduler

A production-inspired distributed job scheduling platform: authentication and
project management, configurable queues, five job types (immediate, delayed,
scheduled, recurring, batch), atomic job claiming across many concurrent
workers, configurable retry strategies with a Dead Letter Queue, and a
live dashboard.

- **Backend:** Python, FastAPI, SQLAlchemy (async), PostgreSQL, Alembic
- **Worker:** Python, `httpx` (talks to the backend over REST — no DB access)
- **Scheduler:** Python (recurring-job spawning, stale-job reclamation)
- **Frontend:** static HTML/CSS/vanilla JS dashboard, no build step

See `docs/ARCHITECTURE.md`, `docs/ER_DIAGRAM.md`, `docs/API_DOCUMENTATION.md`,
and `docs/DESIGN_DECISIONS.md` for the full write-ups.

## Quick start (Docker)

```bash
docker-compose up --build
```

This starts Postgres, the API (`:8000`), the scheduler, two worker replicas
(listening on `default`, `emails`, `reports` queues), and the dashboard
(`:8080`).

- API docs (Swagger UI): http://localhost:8000/docs
- Dashboard: http://localhost:8080

The dashboard defaults to talking to `http://127.0.0.1:8000`. If you deploy
the frontend somewhere else, set `window.SCHEDULER_API_BASE` before the
script tag in `frontend/index.html`, or via a small inline `<script>` block.

## Running locally without Docker

Requires Python 3.12+ and a running PostgreSQL instance.

```bash
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # edit DATABASE_URL if needed
alembic upgrade head    # or just run the API once, which auto-creates tables

uvicorn app.main:app --reload
```

In separate terminals:

```bash
# Scheduler (recurring jobs + stale-job reclamation)
cd backend && python -m app.services.scheduler

# One or more workers
cd worker && pip install -r requirements.txt
python worker.py --queues default,emails --concurrency 5
```

Serve the dashboard with any static file server, e.g.:

```bash
cd frontend && python -m http.server 8080
```

## Running the tests

```bash
cd backend
# tests run against a real Postgres database (not sqlite/mocks)
createdb jobscheduler_test
DATABASE_URL=postgresql+asyncpg://postgres:postgres@localhost:5432/jobscheduler_test \
  pytest -v
```

All 16 tests pass, including `test_concurrent_claims_never_double_assign`,
which fires 8 concurrent claim requests at 10 jobs and asserts every job is
claimed by exactly one worker — the core correctness property of the whole
system.

## Project layout

```
  app/
    core/          # config, DB session, security, auth deps
    models/         # SQLAlchemy models (12 tables) + enums
    schemas/        # Pydantic request/response models

  API_DOCUMENTATION.md

```

## First-run walkthrough

```bash
BASE=http://localhost:8000/api/v1

# 1. Register + log in
curl -X POST $BASE/auth/register -H "Content-Type: application/json" \
  -d '{"email":"you@example.com","password":"changeme123","full_name":"Your Name"}'
TOKEN=$(curl -s -X POST $BASE/auth/login -H "Content-Type: application/json" \
  -d '{"email":"you@example.com","password":"changeme123"}' | python3 -c "import sys,json;print(json.load(sys.stdin)['access_token'])")

# 2. Create an org, project, retry policy, and queue
ORG_ID=$(curl -s -X POST $BASE/organizations -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" -d '{"name":"Acme","slug":"acme"}' | python3 -c "import sys,json;print(json.load(sys.stdin)['id'])")
PROJ_ID=$(curl -s -X POST $BASE/projects -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" -d "{\"organization_id\":\"$ORG_ID\",\"name\":\"Demo\",\"slug\":\"demo\"}" | python3 -c "import sys,json;print(json.load(sys.stdin)['id'])")
Q_ID=$(curl -s -X POST $BASE/queues -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" -d "{\"project_id\":\"$PROJ_ID\",\"name\":\"emails\",\"concurrency_limit\":5}" | python3 -c "import sys,json;print(json.load(sys.stdin)['id'])")

# 3. Submit a job
curl -X POST $BASE/jobs -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d "{\"queue_id\":\"$Q_ID\",\"name\":\"send_welcome_email\",\"payload\":{\"to\":\"user@example.com\"},\"job_type\":\"immediate\"}"

# 4. Watch it get claimed and completed by a running worker, then check:
curl $BASE/queues/$Q_ID/stats -H "Authorization: Bearer $TOKEN"
```

Open the dashboard at http://localhost:8080, register/log in with the same
account, and watch the same data live.
